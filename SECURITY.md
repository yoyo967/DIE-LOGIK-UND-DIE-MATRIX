# Security Policy

## Unterstützte Versionen

| Version | Unterstützt |
| ------- | ------------ |
| main    | ✅ aktiv      |
| ältere Branches | ❌ nicht unterstützt |

---

## Sicherheitslücken melden

### Privates Vulnerability Reporting (Empfohlen)

Für Sicherheitslücken nutze bitte den vertraulichen Kanal:

**GitHub Private Vulnerability Reporting:**
https://github.com/yoyo967/DIE-LOGIK-UND-DIE-MATRIX/security/advisories/new

Dieses Feature ist aktiviert und erlaubt dir, Sicherheitsprobleme direkt und vertraulich an die Maintainer zu melden, ohne sie öffentlich zu machen.

### Antwortzeiten

| Schweregrad | Antwortzeit |
| ----------- | ----------- |
| Kritisch (CVSS 9.0+) | 24 Stunden |
| Hoch (CVSS 7.0-8.9)  | 48 Stunden |
| Mittel (CVSS 4.0-6.9) | 7 Tage |
| Niedrig (CVSS < 4.0) | 14 Tage |

---

## Was wir benötigen

Bitte schließe in deinen Bericht ein:

- Typ der Schwachstelle (z.B. Code Injection, SSRF, XSS, etc.)
- Vollständiger Pfad zur betroffenen Datei oder Komponente
- Schritte zur Reproduktion
- Proof-of-Concept Code (falls vorhanden)
- Auswirkungen: Welche Daten/Systeme sind betroffen?

---

## Compliance-Kontext

Dieses Projekt unterliegt:
- **EU AI Act Artikel 50** (Transparenzpflichten für KI-Systeme)
- **DSGVO** (Datenschutz-Grundverordnung)
- **BSI IT-Grundschutz** (wo anwendbar)

Sicherheitsvorfälle mit Datenschutzbezug werden gemäß Art. 33 DSGVO innerhalb von 72 Stunden an die zuständige Datenschutzbehörde gemeldet.

---

## Kein Bug-Bounty

Aktuell gibt es kein formales Bug-Bounty-Programm. Wir erkennen verantwortungsvolle Disclosure-Meldungen öffentlich in den Release Notes an (mit deiner Genehmigung).

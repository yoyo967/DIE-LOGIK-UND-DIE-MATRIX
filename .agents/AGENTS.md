# Project Rules - DIE LOGIK & DIE MATRIX

## 1. Verifizierbarkeits-Codex (Absolute Truth Rule)
*   **Keine Fiktion:** Es dürfen absolut keine erfundenen historischen Ereignisse, fiktiven Integrationstests oder halluzinierten Meetup-Präsentationen in das Buch, die Spezifikationen oder das Repository geschrieben werden.
*   **Empirische Verifizierbarkeit:** Jedes beschriebene technische Detail, jedes Testergebnis und jeder historische Meilenstein muss empirisch belegbar und faktisch wahr sein.
*   **Ehrliche Metadaten:** Die Dokumentation muss die reale Entwicklungshistorie und die tatsächlichen Gegebenheiten widerspiegeln.
